# 贡献指南

感谢关注本项目。项目处于极早期（Phase 0），当前最有价值的贡献是通过 Issue 参与设计讨论。

## 基本流程

1. 先开 Issue 描述问题或提案，达成一致后再提 PR。
2. PR 保持小而聚焦，一个 PR 只做一件事。
3. 提交信息用中文或英文均可，说清「做了什么、为什么」。

## 代码约定

- Flutter/Dart 代码遵循 `dart format` 默认风格，提交前跑 `flutter analyze` 无警告。
- 新功能对应 PRD 中的功能编号（如 `D4`、`G2`），在 PR 描述中注明。
- 涉及数据模型变更的 PR 需同步更新 `docs/architecture.md`。

## 数据源适配器红线（必读）

本项目的数据源以可插拔适配器（`BookSource` 接口）实现。**以下类型的 PR 一律不予合并：**

- 对接任何以传播盗版内容为主要用途的站点（无论其接口是否「开源」）的适配器；
- 在默认配置、文档或示例中引导用户添加此类站点；
- 绕过上述限制的「预留桩」「注释掉的示例」等变体。

随包默认数据源仅限：Project Gutenberg、Standard Ebooks、维基文库及同等性质的合法公版/授权源。新增默认源需在 Issue 中提供其内容合法性依据。

用户在自己的本地配置中添加什么源是用户自身的行为；但本代码库不为侵权源提供任何形式的内置支持。这条红线同时保护项目本身（避免 DMCA 下架与法律风险）和所有贡献者。

## 行为准则

友善、就事论事、假定善意。
